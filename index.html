Que es la Usabilidad?

La usabilidad es la facilidad con la que un usuario puede navegar en el sitio web,tomando en cuenta caracteristicas como el diseno y organizacion del contenido, y permite que el usuario pueda desplazarse atraves de la pagina de forma sencilla, facil y rapida.

Importancia de la usabilidad

Como la usabilidad se relaciona con la facilidad de uso del sitio web, hace que sea mas sencillo de navegar en el, esto permite que el usuario tenga una mejor experiencia de uso y lograr que mas usuarios visiten la pagina o su tiempo de estadia sea mayor.


Principios de usabilidad

Existen varias reglas que podrian seguirse para mejorar la usabilidad de una pagina web.Entre ellas:


Visibilidad del estado del sistema

Se debe informar a los usuarios sobre el progreso y acciones en la pagina web, para que estos puedan saber el estado de estos. Ejemplo: Los mensajes de confirmacion de una accion como subir un documento, indicar un proceso realizado o una confirmacion de al finalizar formularios.

Conectar con el usuario

Se debe utilizar lenguaje sencillo que los usuarios puedan comprender facilmente sin utilizar tecnicismos, imagenes que cualquiera pueda comprender o iconos reconocibles.

Ejemplo: La pagina de microsoft tiene un lenguaje e imagenes sencillos que se pueden comprender sin problemas y es facil recorrer la web, ademas la descripcion de los botones brinda informacion clara sobre la accion a realizar.

control y libertad de usuario

La pagina web debe permitir que el usuario pueda corregir algun error en caso de que este se equivocase y le sea facil cambiar o eliminar opciones.

Ejemplo: El boton de Eliminar en el carrito de compras de una tienda. En caso de que el usuario se haya equivocado o cambie de opinion y desee quitar el articulo del carrito, puede realizar la accion por medio del boton de 'Eliminar'. Otro ejemplo de esto es al subir un documento, la disponibilidad de botones de editar o eliminar los documentos que se cargaron, da al usuario mas usabilidad de la pagina y no se sienta frustrado por no tener opciones de cambio.

Consistencia y estandares

Se debe seguir estandares para ciertas cosas como por ejemplo los iconos, usar el estandar que la mayoria reconoce para ese icono en lugar de realizar nuevos disenos.

Ejemplo: Los iconos de las redes sociales son facilmente reconocibles por las personas y cualquiera los puede conocer. Si alguien ve los iconos de cada red sabra cual es ya que se usan los que representan el logo de cada red.

Prevencion de errores

Ayudar a que el usuario cometa la menor cantidad de errores y brindar las acciones para que este pueda corregirlos.

Ejemplo: Al registrarse en una pagina, para ciertos campos se comprueban si los datos digitados por el usuario son validos o si hay diferencias entre 2 campos que deberian ser iguales. Se muestra un aviso al usuario para que este conozca el error y pueda correir los campos.

Reconocer antes que recordar

En lugar de hacer que el usuario tenga que recordar los menus o navegacion en la pagina, se debe hacer que sea mas facil para este reconocer los items o manejo de la pagina de una forma sencilla.

Ejemplo: En un formulario donde se tenga que subir un documento, los iconos de la carpeta permiten intuir facilmente esa opcion para cargar el archivo, o bien un signo que le indique al usuario que puede arrastrar los archivos para cargarlos.

Diseno estetico y minimalista

En lugar de hacer que el usuario tenga que recordar los menus o navegacion en la pagina, se debe hacer que sea mas facil para este reconocer los items o manejo de la pagina de una forma sencilla.

Ejemplo: La pagina de disney es sencilla y se puede navegar facilmente por el sitio.

Ayudar a los usuarios a reconocer y diagnosticar errores

Brindarle al usuario una forma de que este sepa que es el error y que podria hacer al respecto. Asi puede orientarse mejor sobre lo que sucede en la pagina.

Ejemplo: Varias paginas personalizan un indicador del error 404, e incluyen mensajes o disenos creativos para informarle al usuario el problema. Algunas anaden una opcion de volver atras o una barra de busqueda.

Ayuda y documentacion

Es bueno que haya una guia de funcionamiento del sitio que el usuario pueda consultar facilmente como un boton de ayuda, seccion de preguntas frecuentes o guias de uso o tutoriales. Con esto el usuario podria facilitarse el uso de la pagina que visita.

Ejemplo: Algunos sitios tienen guias o tutoriales en forma de mensajes que brindan informacion sobre las funciones de la pagina y como usarlas.
